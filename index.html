网络体系结构

协议层次

  • 协议

    • 同一层上对等实体之间所交换的数据包或报文的格式和含义

  • 服务

    • 某一层向它上一层提供的一组原语（操作）

  • 设计要点

    • 可靠性

    • 可扩展性

    • 安全性

参考模型

  • OSI 7层模型

    • 应用层->报文

      • 各种网络程序及协议

    • 表示层

      • 关注传递信息的语法和语义

    • 会话层

      • 允许不同机器上的用户建立会话

    • 传输层->报文段

      • 在应用程序端点之间拆分并传送应用层报文

    • 网络层->数据报

      • 控制子网的运行：将数据包从源端路由到接收方

    • 数据链路层->帧

      • 讲一个原始的传输设施转变成一条没有漏检传输错误的线路

    • 物理层

      • 关注在一条通信信道上传输原始比特

  • TCP/IP 4层模型

网络核心

分组交换

  • 存储转发传输报文

  • 时延、丢包和吞吐量

    • 时延

      • 处理时延

        • 检查分组首部和决定将该分组导向何处所用的时间

      • 排队时延

        • 在队列中等待向链路传输的时间

      • 传输时延

        • 路由器将分组推向链路所需要的时间：L/R

      • 传播时延

        • 一个比特从链路的起点到终点所需要的时间

    • 丢包

      • 流量强度（La/R）>1时会出现丢包

    • 吞吐量

      • 平均吞吐量：F/T bps

      • 简单情况下，链路吞吐量收瓶颈链路影响

      • 多人使用时吞吐量会被均分

  • 报文按需使用资源（缓存、链路传输速率

  • 转发表和路由选择协议

电路交换

  • 通信时创建一条专用的端到端连接

  • 预留带宽，总带宽被多条电路均分

  • 建立连接后以恒定速率传输数据

  • 复用技术

    • 频分复用

    • 时分复用

比较

  • 电路交换实时性好——电话、视频会议，适合人少的情况下用

  • 分组交换有更好的带宽共享、更简单高效，适合人多的情况用

Docker网络

2020-08-29
知识点总结

网络驱动

  • bridge：默认的网络驱动。当应用程序需要在容器中独立运行时通常使用Bridge网络。
  • host：对于独立容器，删除容器和主机之间的网络隔离，让容器直接使用主机的网络。
  • overlay：覆盖网络将多个Docker守护进程连接在一起，使集群服务能够相互通信。
  • macvlan：允许为容器分配MAC地址，让它在网络中像一台物理设备。
  • none：禁用所有网络。

Kubernetes对象

理解Kubernetes对象

在 Kubernetes 系统中，Kubernetes 对象 是持久化的实体。Kubernetes 使用这些实体去表示整个集群的状态。特别地，它们描述了如下信息：

  • 哪些容器化应用在运行（以及在哪个 Node 上）
  • 可以被应用使用的资源
  • 关于应用运行时表现的策略，比如重启策略、升级策略，以及容错策略
    Kubernetes 对象是 “目标性记录” —— 一旦创建对象，Kubernetes 系统将持续工作以确保对象存在。通过创建对象，本质上是在告知 Kubernetes 系统，所需要的集群工作负载看起来是什么样子的，这就是 Kubernetes 集群的 期望状态（Desired State）。

操作 Kubernetes 对象 —— 无论是创建、修改，或者删除 —— 需要使用 Kubernetes API。比如，当使用 kubectl 命令行接口时，CLI 会执行必要的 Kubernetes API 调用，也可以在程序中使用 客户端库 直接调用 Kubernetes API。

vnode与inode的区别

文章整理自 FreeBSD docs 和 StackOverflow

vnode

名称

vnode —— 文件或目录的内部表示

概要

描述

  • vnode是UNIX中所有文件活动的焦点。
  • vnode节点由struct vnode声明。
  • 每个活动文件、当前目录、挂载文件、文本文件和根目录都分配了一个vnode节点。
  • vnode最初是为FreeBSD创建的，因为需要使用的不同类型的文件系统（例如UFS、NFS等）激增。
  • Vnode旨在为所有可能的文件系统提供抽象，以便OS可以与它们进行对接，从而让内核方法不必兼容每个特定的文件系统；内核只需要知道如何与Vnode进行交互。
  • Vnode抽象在内存中，可以与UFS、ext4等文件系统进行交互；而inode存储在磁盘上，只能用于特定的文件系统。
  • inode节点包含文件大小、拥有者、指向地址块的指针等元数据。
  • vnode仅包含在文件生命周期中不会改变的一些属性，因此如果想获得更多信息请使用inode作为参考。
